# Структура контента

Контент организован по частям и главам. Главы хранятся в отдельных markdown файлах, которые автоматически загружаются при сборке.

## Структура

```
content/
├── index.ts              # Конфигурация частей и глав
├── loader.ts             # Утилита для загрузки markdown файлов
├── part-1/
│   ├── chapter-1.md     # Глава 1
│   ├── chapter-2.md     # Глава 2
│   └── chapter-3.md     # Глава 3
├── part-2/
│   ├── chapter-1.md     # Глава 1
│   └── chapter-2.md     # Глава 2
└── ...
```

## Как добавить новую часть

1. Создайте папку `part-N/`
2. Создайте markdown файлы для глав: `chapter-1.md`, `chapter-2.md`, и т.д.
3. Добавьте конфигурацию в `index.ts`:

```typescript
{
  id: 'part-N',
  title: 'Название части',
  folder: 'part-N',
  chapters: [
    {
      id: 'chapter-N-1',
      title: 'Название главы',
      file: 'chapter-1.md',
    },
    // ...
  ],
}
```

## Формат markdown файлов

Каждый файл должен начинаться с заголовка `#`:

```markdown
# Заголовок главы

Содержимое главы в markdown формате...

## Подзаголовок

Еще контент...
```

Заголовок из первой строки автоматически используется как название главы, если не указан в конфигурации.

## Преимущества

- ✅ **Чистые markdown файлы** - легко редактировать в любом редакторе
- ✅ **Автоматическая загрузка** - не нужно создавать промежуточные файлы
- ✅ **Масштабируемость** - легко добавлять новые части и главы
- ✅ **Типобезопасность** - TypeScript проверяет структуру
- ✅ **Разделение контента и метаданных** - конфигурация отдельно от текста

## Как это работает

1. `loader.ts` читает markdown файлы из папок с помощью `fs.readFileSync`
2. Извлекает заголовок из первой строки markdown (если не указан в конфигурации)
3. Строит структуру `ReaderContent` с частями и главами
4. Все происходит на этапе сборки (build time)
